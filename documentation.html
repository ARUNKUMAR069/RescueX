Project Overview

RescueGrid is an advanced disaster prediction and prevention system designed to help communities prepare for and mitigate the effects of natural disasters. Using real-time weather data, historical patterns, and machine learning algorithms, RescueGrid can predict a wide range of potential disasters and provide actionable prevention measures.

The system is built with a focus on:

  • Accuracy: Utilizing advanced prediction algorithms and continuous learning from past data
  • Comprehensiveness: Covering multiple disaster types from floods and heat waves to earthquakes and wildfires
  • Accessibility: Easy-to-understand interface available on any device with internet access
  • Actionability: Practical prevention measures tailored to each specific disaster scenario
  • Adaptability: Self-improving system that learns from feedback and actual outcomes

Whether you're an individual concerned about local risks, an emergency management agency preparing response plans, or a business ensuring operational continuity, RescueGrid provides valuable insights and recommendations to enhance safety and resilience.

Why RescueGrid Matters

Natural disasters affect millions of people worldwide each year, causing significant loss of life, property damage, and economic disruption. Early prediction and effective preventive measures can dramatically reduce these impacts, saving lives and protecting communities. RescueGrid bridges the gap between complex meteorological and geological data and practical, actionable information for everyday users.

Geolocation & Air Quality Monitoring

RescueGrid now provides advanced location-based features and comprehensive air quality monitoring to help users stay aware of potential hazards in their immediate environment.

Automatic Location Detection

The system now automatically detects your current location when you open the application, providing immediate disaster risk assessment for your area without manual input.

  • Uses browser's geolocation API for precise positioning
  • Converts coordinates to location names automatically
  • Provides instant risk analysis on page load
  • Fallback to manual search if permission is denied

Air Quality Index Monitoring

Comprehensive air quality monitoring now included to detect potential air quality emergencies, including gas leaks and industrial incidents.

  • Real-time AQI data with color-coded risk levels
  • Detection of potentially hazardous air conditions
  • Specific alerts for extreme pollution events
  • Gas leak and chemical release early warning

Air Quality Index (AQI) Scale

AQI Range Category Health Implications Recommended Actions
0-50 Good Air quality is satisfactory with little or no risk No special precautions needed
51-100 Moderate Acceptable, but may cause minor concern for sensitive individuals People with respiratory or heart conditions should consider limiting prolonged outdoor exertion
101-150 Unhealthy for Sensitive Groups Members of sensitive groups may experience health effects Children, elderly, and people with respiratory or heart conditions should limit outdoor exertion
151-200 Unhealthy Everyone may begin to experience health effects Avoid prolonged outdoor exertion; everyone should limit outdoor activity
201-300 Very Unhealthy Health alert: everyone may experience more serious health effects Avoid all outdoor physical activity; remain indoors with filtered air
301+ Hazardous Emergency conditions; entire population likely affected Stay indoors with air purification; follow emergency directives; check for evacuation orders

Gas Leak Detection

Extremely high AQI values (300+) in combination with specific air quality parameters can indicate potential gas leaks or chemical releases, especially when they occur suddenly in localized areas.

Emergency Actions for Gas Leak Alerts:
  • Evacuate the area immediately
  • Do not use electrical devices, matches, or create sparks
  • Call emergency services from a safe location
  • Do not return until authorities declare the area safe
  • Seek medical attention if experiencing symptoms like headache, dizziness, nausea, or difficulty breathing

System Architecture

Architecture Overview

RescueGrid is built on a modern, scalable architecture designed for reliability, speed, and maintainability.

Key Components

  • Frontend: Responsive HTML/CSS/JavaScript interface with Tailwind CSS for styling
  • Backend API: FastAPI (Python) providing RESTful endpoints for weather data, predictions, and recommendations
  • Prediction Engine: Core Python modules for disaster analysis and forecasting
  • Data Storage: SQLite database storing prediction history and user feedback
  • External APIs: Integration with weather data providers for real-time information

Prediction Parameters

RescueGrid analyzes numerous environmental parameters to make predictions:

Parameter Used For Valid Range

Machine Learning & Continuous Improvement

RescueGrid employs a self-improving system that learns from user feedback and historical prediction accuracy.

Learning Capabilities

How RescueGrid Learns

  1. Prediction Storage:

    Every prediction is stored in the database with all relevant parameters and outcomes.

  2. User Feedback Collection:

    Users can rate prediction accuracy as "Highly Accurate," "Somewhat Accurate," or "Not Accurate."

  3. Coefficient Adjustment:

    Learning coefficients for different disaster types are automatically adjusted based on feedback.

  4. Pattern Recognition:

    The system identifies patterns in successful vs. unsuccessful predictions.

  5. Model Recalibration:

    Prediction models are periodically recalibrated to incorporate learned patterns.

Learning Coefficients

RescueGrid uses a set of learning coefficients that adjust the weight given to different factors in prediction algorithms:

self.learning_coefficients = {
    'flood': 1.0,
    'heat_wave': 1.0,
    'storm': 1.0,
    'wildfire': 1.0,
    'tornado': 1.0,
    'earthquake': 1.0,
    'air_quality': 1.0
}

These coefficients are adjusted over time based on feedback:

  • Coefficients increase when predictions are accurate
  • Coefficients decrease when predictions are inaccurate
  • Adjustments are proportional to the level of accuracy reported

Impact on Predictions

Learning improves predictions in several ways:

Regional Calibration

Adjusts prediction thresholds based on how different regions respond to similar weather conditions.

Seasonal Adaptation

Recognizes how the same parameters may have different impacts depending on the season.

Parameter Weighting

Adjusts the relative importance of different weather and environmental factors.

False Positive Reduction

Learns to avoid triggering alerts in conditions that historically did not lead to disasters.

API Documentation

RescueGrid provides a comprehensive REST API for developers who want to integrate disaster predictions into their own applications.

Endpoints

Endpoint Method Description Parameters
/api/predict GET Get disaster predictions for a location location (required): City name
/api/weather GET Get current weather data location (required): City name or coordinates
/api/forecast GET Get weather forecast location (required), days (optional): 1-3
/api/history GET Get prediction history limit (optional): Maximum records to return
/api/feedback POST Submit feedback on predictions prediction_id, feedback_type, feedback_value

Installation Guide

Follow these steps to set up your own instance of RescueGrid.

Prerequisites

  • Python 3.8 or higher
  • pip (Python package manager)
  • Git
  • WeatherAPI account for API key

Installation Steps

  1. Clone the Repository 
    git clone https://github.com/yourusername/RescueGrid_KTK.git
cd RescueGrid_KTK/Recomend
  2. Set Up Virtual Environment (Optional but Recommended) 
    python -m venv venv
# On Windows
venv\Scripts\activate
# On macOS/Linux
source venv/bin/activate
  3. Install Dependencies 
    pip install -r requirements.txt
  4. Configure Environment Variables

    Create a .env file in the project root with the following content:

    WEATHER_API_KEY=your_weather_api_key
DATABASE_URL=sqlite:///./rescuegrid.db
DEBUG=True
  5. Initialize the Database 
    python initialize_db.py
  6. Run the Application 
    uvicorn app:app --reload

    Your RescueGrid instance will be available at http://localhost:8000

Troubleshooting

Common Issues

  • API Key Issues:

    If you see errors related to the Weather API, check that your API key is correct and properly set in the .env file.

  • Database Errors:

    If database errors occur, try removing the rescuegrid.db file and re-running initialize_db.py.

  • Dependency Conflicts:

    If you encounter package conflicts, try creating a fresh virtual environment and reinstalling dependencies.

  • Port Already in Use:

    If port 8000 is already in use, specify a different port using uvicorn app:app --port 8001.

Production Deployment

For production environments, additional steps are recommended:

  • Use a production-grade ASGI server like Gunicorn with Uvicorn workers
  • Set up a reverse proxy (Nginx or Apache) in front of the application
  • Configure HTTPS using a valid SSL certificate
  • Set DEBUG=False in your production environment variables
  • Consider using a more robust database like PostgreSQL for high-traffic deployments

Docker Deployment

RescueGrid can also be deployed using Docker:

# Build the Docker image
docker build -t rescuegrid .

# Run the container
docker run -d -p 8000:8000 --name rescuegrid-instance \
  -e WEATHER_API_KEY=your_weather_api_key \
  rescuegrid
